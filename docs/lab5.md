# Лабораторна робота №5

## Тестування програмного забезпечення

### 1. Unit-тестування

Було реалізовано unit-тести для сервісного шару (`task.service.ts`), які перевіряють:

- створення задачі з дефолтним статусом
- валідацію вхідних даних
- зміну стану задачі
- обробку помилок

### 2. Інтеграційне тестування

Інтеграційні тести перевіряють роботу API з реальною базою даних:

- тестується endpoint `GET /tasks`
- використовується тестова PostgreSQL база
- Prisma використовується як ORM

### 3. E2E тестування

E2E тест охоплює повний бізнес-сценарій:

- створення користувача
- створення проєкту
- створення задачі
- отримання списку задач

Тест запускає реальний HTTP сервер та працює з реальною БД.

### 4. Мутаційне тестування

Для мутаційного тестування використано **Stryker** з Jest runner.

Результати:

- Mutation score: **49.41%**
- Covered code: **82.35%**
- Killed mutants: 41
- Survived mutants: 9
- No coverage mutants: 34

Файли `seed.ts` та `server.ts` не покриті тестами, що є допустимим, оскільки вони не містять бізнес-логіки.

### 5. Висновки

Мутаційне тестування показало, що основна бізнес-логіка добре покрита тестами, проте:

- маршрути можна тестувати детальніше
- варто перевіряти структуру відповідей API
- деякі граничні випадки не покриті
