# Лабораторна робота №6

## Розгортання програмного забезпечення

### 1. Continuous Integration

Було налаштовано CI за допомогою GitHub Actions.
Pipeline автоматично запускається при push або pull request у гілку main.

CI включає:

- встановлення залежностей
- перевірку форматування (Prettier)
- статичний аналіз коду (ESLint)
- запуск unit, integration та e2e тестів
- підключення PostgreSQL через Docker service

Merge pull request неможливий без успішного проходження CI.

### 2. Continuous Deployment

Для CD використовується Docker.
Після змін у гілці main можливо автоматично розгорнути нову версію застосунку.

Було підготовлено Dockerfile для контейнеризації застосунку.
CD workflow може бути розширений для деплою на хмарні сервіси (Render, Railway, VPS).

### 3. Доступність сервісу

Застосунок може бути запущений у Docker-контейнері та доступний через HTTP.

### 4. Висновки

CI/CD процес дозволяє автоматично перевіряти якість коду, запускати тести та гарантує стабільність перед розгортанням.
