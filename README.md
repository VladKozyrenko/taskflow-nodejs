# TaskFlow

## Загальна інформація

Проєкт виконується індивідуально в рамках курсу
«Розробка програмного забезпечення на платформі Node.js».

## Опис ідеї

TaskFlow — це backend-застосунок для управління задачами та проєктами,
який надає REST API для взаємодії з клієнтськими застосунками.

Система дозволяє створювати проєкти, додавати задачі,
керувати їх статусами та отримувати актуальний стан роботи.

## Мета проєкту

- Закріпити практичні навички роботи з Node.js
- Ознайомитися з архітектурою серверних застосунків
- Реалізувати повний цикл розробки ПЗ: від ідеї до деплою

## Основний функціонал

- Управління користувачами
- Створення та редагування проєктів
- Управління задачами
- Зміна статусів задач
- REST API для інтеграції

## Технологічний стек (попередній)

- Node.js
- Express.js
- TypeScript
- MongoDB
- Jest
- GitHub Actions

## Розгортання (Lab 6)

У проєкті використовується GitHub Actions для реалізації CI/CD.

### CI

При кожному коміті або Pull Request:

- виконується форматування коду за допомогою Prettier
- запускається ESLint для статичного аналізу коду
- виконуються unit-, integration- та E2E-тести
- перевіряється, що проєкт коректно збирається та проходить усі перевірки

Це гарантує стабільність коду та відповідність стандартам перед злиттям змін.

### CD

При кожному коміті в гілку main:

- автоматично запускається CD pipeline
- збирається Docker-образ застосунку
- застосунок готовий до розгортання на будь-якому хостингу, що підтримує Docker

Проєкт контейнеризований та може бути легко розгорнутий у продакшн-середовищі.

### Docker

Для підготовки до розгортання використовується Docker.
У корені репозиторію присутній Dockerfile, який:

- встановлює залежності
- генерує Prisma Client
- запускає сервер застосунку
